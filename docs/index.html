CinConecta

Projeto de gestão de estoque de produtos com autenticação

Este projeto implementa um sistema de gerenciamento de produtos, com funcionalidades como pesquisa, adição, atualização, remoção de produtos e autenticação de usuários. O sistema utiliza um banco de dados SQLite para persistir dados e foi desenvolvido utilizando conceitos de Programação Orientada a Objetos (OOP).

Vídeo resumido do projeto

Arquivo: controlador.cpp

Nosso controlador segue uma arquitetura RESTful e é responsável por gerenciar requisições HTTP na aplicação. Ele é desenvolvido em C++ com a biblioteca Crow, garantindo a comunicação entre o cliente e o banco de dados SQLite.

O controlador executa funções como:

  • Recuperação de produtos (por usuário ou consulta específica).
  • Adição, remoção e atualização de produtos.
  • Registro, autenticação e logout de usuários.

Separamos a lógica de requisição, manipulação de dados e resposta JSON. Além disso, utilizamos hashSimples para validação segura de senhas.

Arquivo: controlador.h

Define as funções essenciais para o controle de produtos e usuários em nosso sistema. Ele funciona como uma ponte entre as requisições HTTP, feitas com a biblioteca Crow, e o banco de dados SQLite, garantindo que possamos gerenciar os dados de forma eficiente.

Arquivo: dbmanager.cpp

O gerenciamento de produtos e usuários é realizado com funções que permitem:

  • Busca, adição, atualização e remoção de produtos.
  • Registro de usuários e verificação de credenciais.
  • Geração, verificação e revogação de tokens.

Arquivo: hashing.h

Este arquivo lida com a geração e verificação de hashes:

  • Geração de hash utilizando um algoritmo simples com fator primo.
  • Verificação de hash para autenticação segura de senhas.

Arquivo: horarios.h

Funções principais para manipulação de datas e horários:

  • Obtenção da hora atual no formato "YYYY-MM-DD HH:MM:SS".
  • Conversão de string para estrutura de tempo std::tm.
  • Comparação de datas para verificação de validade de tokens.

Arquivo: produto.h

A estrutura Produto encapsula atributos relacionados a um produto, garantindo um bloco lógico coeso. Um construtor facilita a inicialização dos atributos.

Arquivo: router.h

Contém a definição das rotas utilizadas pelo servidor para manipulação de produtos e usuários:

  • Rotas GET, POST, DELETE e PUT para manipulação de produtos.
  • Rotas para registro, login e logout de usuários.

Arquivo: main.cpp

Principais funcionalidades:

  • Inicialização do servidor com a biblioteca Crow.
  • Conexão com o banco de dados SQLite.
  • Registro de rotas e encerramento seguro do servidor.

PASTA: token.h

Funções principais:

  • Geração de números aleatórios para criação de tokens seguros.
  • Uso de std::stringstream para formatação do token.
  • Tokens com 32 caracteres hexadecimais, garantindo unicidade.
